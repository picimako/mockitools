Can perform conversion of InOrder.verify() call chains to the following approaches:
  • to Mockito.verify(),
  • to an InOrder specific BDDMockito.then() using a newly created an InOrder local variable,
  • to BDDMockito.then() without an InOrder, practically omitting the source InOrder variable.

The intention is available on InOrder.verify() when it is followed by a method call on the mock object, when the user wants to convert only a single verification call, and also available when the users select one or more InOrder.verify() call chains that they want to convert.

Call chains when there is no separate InOrder local variable is created, but Mockito.inOrder(mockObject) is called inline, is not supported at the moment.

Conversion to Mockito.verify() is available only when the Enforce conventions inspection is disabled, or it enforces org.mockito.Mockito, otherwise the two BDDMockito.then() specific conversions are available.

In any other case, the option list displays a No action available message.
