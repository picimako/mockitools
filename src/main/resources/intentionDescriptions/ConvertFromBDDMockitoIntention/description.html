Can perform conversion of BDDMockito.then().should() call chains to the following approaches:
  • to Mockito.verify(),
  • to an InOrder specific BDDMockito.then() using a newly created an InOrder local variable, if there is none specified in the should() call yet,
  • to InOrder.verify() by either creating a new InOrder variable, or reusing the one specified in the should() call

The intention is available on BDDMockito.then() when it is followed by should() and a method call on the mock object.

Conversion to Mockito.verify() and InOrder.verify() are available only when the Enforce conventions inspection is disabled, or it doesn't enforce org.mockito.BDDMockito.

The InOrder specific BDDMockito.then() is available only when the source BDDMockito.then().should() call chain doesn't contain an InOrder already.
