This inspection will report any call to Mockito.reset(), regardless of its location in the test method. Of course there may be cases where calling reset() is acceptable or even necessary, this inspection doesn't take into account those cases.

Based on Mockito's documentation on resetting mocks

"Smart Mockito users hardly use this feature because they know it could be a sign of poor tests. Normally, you don't need to reset your mocks, just create new mocks for each test method."
    
@Test
void testMethod() {
    //mock setup
    MockObject mock = Mockito.mock(MockObject.class);
    when(mock.doSomething())...
    reset(mock); //reset() is called
    //another mock setup
    when(mock.doSomethingElse())...
}
