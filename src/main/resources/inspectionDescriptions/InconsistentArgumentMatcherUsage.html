Argument matchers are used incorrectly when, during stubbing, matcher and non-matcher arguments are used in a mixed manner. That is not permitted by Mockito. For example: 
    Mockito.when(mock.method(eq("some string"), 25)).thenReturn(10);
or 
    Mockito.when(mock.method("some string", geq(25))).thenReturn(10);
As per Mockito documentation: If you are using argument matchers, all arguments have to be provided by matchers.

The following ways of stubbing are supported:

  • Mockito.when()
  • BDDMockito.given()
  • Mockito.do...().when()
  • matchers in AdditionalMatchers

You can find more information in the official documents: Argument matchers documentation, ArgumentMatchers javadoc
